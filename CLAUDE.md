# Рабочее место 1С-разработчика

## Структура

```
E:\programming\
├── .claude\skills\     ← скиллы (глобальные)
├── _rules\             ← правила (глобальные)
├── _roles\
│   ├── analyst.md      ← роль аналитика (самодостаточная, включает свои правила)
│   └── developer.md    ← роль разработчика (самодостаточная, включает свои правила)
├── _templates\
│   └── CLAUDE-project.md  ← шаблон CLAUDE.md для нового проекта
└── clients\            ← все клиентские проекты (gitignored целиком)
    ├── reviline\
    ├── sostra\
    ├── krzht\
    ├── tradeparts\
    └── big\
```

## Проекты

| Псевдоним  | Описание          |
|------------|-------------------|
| sostra     | Клиент Sostra     |
| krzht      | Клиент Krzht      |
| tradeparts | Клиент Tradeparts |
| big        | Клиент Big        |
| reviline   | Клиент Reviline   |

## Общие соглашения

- Среда: Windows, 1С 8.3, OneScript установлен, базы опубликованы на IIS
- Перед работой с проектом читать `CLAUDE.md` в папке проекта
- Типовую конфигурацию (`mainConf\`) не модифицировать — только чтение
- Все изменения только через расширения (CFE) или внешние обработки (EPF/ERF)
- Задачи: `tasks\active\T-XXXX_название.md`

## Протокол ролей

В начале сессии уточни роль, если она не задана: **аналитик** или **разработчик**.
Затем прочитай соответствующий файл роли и строго соблюдай его ограничения:

- **Аналитик** → `E:\programming\_roles\analyst.md`
- **Разработчик** → `E:\programming\_roles\developer.md`

> Проекты находятся в `clients\<псевдоним>\`. VS Code открывать на папке проекта (`clients\reviline\`), не на `clients\`.

## Параллельная работа двух ролей

Одна сессия = один контекст. Два окна VS Code на одной папке проекта (не на папке роли — иначе пропадёт `.mcp.json`):

- Окно A: «ты аналитик»
- Окно B: «ты разработчик»

Обмен между ролями — **только через файлы** `tasks\active\T-XXXX_*.md`.
Переключить роль в том же окне — через `/clear`.
