# 1C Workspace — Автоматизированное рабочее место 1С-разработчика

Скелет управляемой среды для команды 1С-разработки и интеграции ИИ-ассистентов (Claude Code / MCP). 
Решение содержит стандартизированные роли, правила разработки и автоматическую увязку ИИ-скиллов для быстрого старта на проектах.

## Быстрый старт

Развертывание рабочего места выполняется одной командой. Вы можете склонировать репозиторий в **любую удобную папку** на вашем компьютере. 



```powershell
# Клонируем репозиторий вместе со всеми подмодулями скиллов
git clone --recurse-submodules https://github.com/1c-nik/1c-workspace.git "Путь\К\Вашей\Папке"

# Переходим в созданную папку
cd "Путь\К\Вашей\Папке"

# Запускаем скрипт первичной настройки (права администратора не требуются)
.\setup.ps1

```

После завершения скрипта откройте рабочее пространство в VS Code:

```powershell
code .

```

> **Установка расширений:** При первом запуске VS Code заметит файл конфигурации проекта и предложит автоматически установить рекомендованные плагины для работы с 1С (BSL) и ИИ. Просто нажмите **Install** во всплывающем окне внизу справа.

## Что внутри и как это работает

### 1. Локальная автоматизация (`setup.ps1` и `.vscode`)

Скрипт выполняет «бесшовную» настройку связи:

* Проверяет и фиксирует состояние внешних ИИ-скиллов из подмодуля.
* Создает файловую связь (Junction) `.claude\skills\` $\rightarrow$ `claude-code-skills-1c\.claude\skills\`. ИИ-агент видит скиллы по стандартному пути, при этом они не дублируются на диске и легко обновляются.
* Файл `.vscode/extensions.json` сам разворачивает в редакторе среду поддержки языка 1С (BSL) при старте.

### 2. Контекстные ИИ-роли (`_roles\`)

Разделяют логику взаимодействия с ассистентом на два изолированных контура:

* **Аналитик:** Работает в режиме расследования ошибок, анализа метаданных и построения запросов. Не имеет прав на изменение кода.
* **Разработчик:** Подключается по готовому ТЗ. Реализует логику в расширениях (CFE) и внешних обработках (EPF/ERF).
* Обмен информацией между окнами аналитика и разработчика происходит строго через файлы задач в формате `tasks\active\T-XXXX_название.md`.

### 3. Скиллы (`claude-code-skills-1c\`)

Централизованный репозиторий контекстных умений ИИ от Desko77 (86 готовых скиллов).
Для синхронизации и получения обновлений от автора достаточно выполнить:

```bash
cd claude-code-skills-1c
git pull

```

## Добавление нового проекта (клиента)

1. Создайте изолированную папку: `clients\<имя_клиента>\`
2. Скопируйте шаблон: `_templates\CLAUDE-project.md` $\rightarrow$ `clients\<имя_клиента>\CLAUDE.md`
3. Заполните специфику проекта (порты MCP-сервера, особенности конфигурации).
4. **Важно:** Всегда открывайте в VS Code конечную папку клиента (`clients\<имя_клиента>\`), а не весь каталог воркспейса. Это изолирует контекст ИИ, чтобы он не путал метаданные разных баз.

## Технологические соглашения

* **Безопасность данных:** Папка `clients/` находится в глобальном `.gitignore`. Исходники клиентов, конфигурации и коммерческая информация никогда не попадут в ваш публичный или приватный репозиторий воркспейса.
* **Чистота конфигурации:** Основная конфигурация клиента (`mainConf/`) открывается ИИ строго в режиме **Read-Only**. Любая доработка ведется исключительно через механизмы расширений (`.cfe`) или внешних отчетов/обработок.
